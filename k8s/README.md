## ğŸš€ Deployment Steps

### 1. Deploy Database & Storage
First, bring up the database layer.

```bash
# create storage and mongo deployment
kubectl apply -f mongo-pvc.yaml
kubectl apply -f mongo.yaml
```

Wait for MongoDB to be ready:
```bash
kubectl wait --for=condition=ready pod -l app=mongo --timeout=60s
```

### 2. Seed the Database
Populate MongoDB with initial movie data.

```bash
# create configmap with seed script
kubectl apply -f mongo-seed-configmap.yaml

# run the seed job
kubectl apply -f mongo-seed-job.yaml
```

Verify the job completed:
```bash
kubectl get jobs -w
# Wait for COMPLETIONS 1/1
```

### 3. Deploy Application
Deploy the backend and frontend services.

```bash
kubectl apply -f movie-manager-backend.yaml
kubectl apply -f movie-manager-frontend.yaml
```

### 4. Expose via Ingress (ALB)
Create the Ingress resource to route traffic from the internet.

```bash
kubectl apply -f movie-manager-ingress.yaml
```

## ğŸ” Verification

### Get Ingress URL
It may take a few minutes for AWS to provision the load balancer.

```bash
kubectl get ingress movie-manager-ingress
```
Copy the `ADDRESS` (e.g., `k8s-default-movieman-....us-east-1.elb.amazonaws.com`) and open it in your browser.

### Troubleshooting
If the address is empty or not working:
1. Ensure the **AWS Load Balancer Controller** is installed (see `infra/eks/README.md`).
2. Check logs:
   ```bash
   kubectl logs -n kube-system -l app.kubernetes.io/name=aws-load-balancer-controller
   ```
